# Reproducibility Guide

This guide describes how to reproduce the current leakage-aware single-cell chemical perturbation benchmark outputs.

## Environment

Use the project virtual environment when available:

```bash
.venv/bin/python --version
```

The completed v8 resource assembly requires `pandas`, `numpy`, `matplotlib`, `anndata`, `RDKit`, and `scikit-learn`. Deep perturbation models and frozen foundation embeddings require additional dependencies that are not bundled in the current environment.

## Rebuild Existing Baseline Summaries

Existing OpenProblems and Sci-Plex 3 baseline outputs were generated by the scripts in `scripts/`. The current v8 add-on does not alter those split definitions or model outputs.

To rebuild the standardized deep-model-panel result tables from existing outputs:

```bash
.venv/bin/python scripts/run_deep_model_panel_smoke.py
```

This writes:

```text
results/deep_model_panel/all_model_metrics_long.csv
results/deep_model_panel/all_model_leakage_audit_long.csv
results/deep_model_panel/model_rank_stability.csv
results/deep_model_panel/random_to_strict_contrasts.csv
results/deep_model_panel/bootstrap_ci.csv
```

## Run Quality Control

```bash
.venv/bin/python scripts/qc_deep_model_panel.py
```

The QC checks cover metric behavior on toy data, split integrity for available leakage columns, no-test-tuning configuration, and output completeness.

## Rebuild v8 Figures

```bash
.venv/bin/python scripts/plot_deep_model_panel_figures.py
```

Figures and source data are written to:

```text
results/deep_model_panel/figures/
results/deep_model_panel/source_data/
```

## Deep Model Panel Workflow

The planned workflow is:

1. Run one-seed smoke tests for each adapter.
2. Run 10-seed pilots if the smoke test passes.
3. Run 30-seed main evaluations.
4. Record all failures as status rows rather than dropping model/split/seed combinations.

All model adapters must use the existing split manifests and must not tune against test records.

## Public Release Checklist

Before submission, replace placeholder URLs with a public GitHub repository and Zenodo DOI. Do not invent accession numbers or DOIs.
